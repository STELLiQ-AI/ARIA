# Database Package — Room Persistence Layer

## Responsibility
Persists session data, transcript segments, and performance metrics using Room ORM
with Write-Ahead Logging (WAL) for concurrent read/write during recording.

## Schema (Version 3)

### `aar_sessions` (entity: AARSession)
Primary session record with transcript, AAR summary fields, and metadata.

| Column | Type | Notes |
|--------|------|-------|
| uuid | TEXT PK | Generated by SessionController at INITIALIZING |
| session_title | TEXT | LLM-generated title |
| transcript_full | TEXT | Complete accumulated transcript |
| what_was_planned | TEXT | TC 7-0.1 Field 1 |
| what_happened | TEXT | TC 7-0.1 Field 2 |
| why_it_happened | TEXT | TC 7-0.1 Field 3 |
| how_to_improve | TEXT | TC 7-0.1 Field 4 |
| completeness_score | REAL | 0.0-1.0 (added in migration v2→v3) |
| started_at_epoch_ms | INTEGER | Session start time |
| ended_at_epoch_ms | INTEGER | Session end time |
| duration_ms | INTEGER | Computed duration |
| llm_parse_success | INTEGER | Boolean — did JSON parse succeed? |
| aar_json | TEXT | Raw LLM output for debugging |
| device_model | TEXT | Build.MODEL |
| app_version | TEXT | Constants.APP_VERSION |

### `transcript_segments` (entity: TranscriptSegment)
Per-chunk Whisper output with speaker attribution and performance metrics.

### `pipeline_metrics` (entity: PipelineMetric)
Performance telemetry per session (TTFT, TPS, RTF, RAM usage).

## Files

| File | Responsibility |
|------|---------------|
| `AARDatabase.java` | Room database singleton. WAL mode. Migration v2→v3. Fallback to destructive for unknown jumps. |
| `AARRepository.java` | Repository pattern. Async writes via single-thread executor. Provides lifecycle methods. |
| `dao/AARSessionDao.java` | CRUD for sessions. `insertOrUpdate()` via `@Insert(onConflict = REPLACE)`. |
| `dao/TranscriptSegmentDao.java` | Insert/query segments by session UUID. |
| `dao/PipelineMetricDao.java` | Insert/query performance metrics. |
| `entity/AARSession.java` | Room entity — field mapping to aar_sessions table. |
| `entity/TranscriptSegment.java` | Room entity — per-chunk transcript data. |
| `entity/PipelineMetric.java` | Room entity — performance telemetry. |

## Key Patterns

### Migration v2 → v3
```java
static final Migration MIGRATION_2_3 = new Migration(2, 3) {
    @Override
    public void migrate(SupportSQLiteDatabase database) {
        database.execSQL("ALTER TABLE aar_sessions ADD COLUMN completeness_score REAL NOT NULL DEFAULT 0.0");
    }
};
```
Added BEFORE `fallbackToDestructiveMigration()` to preserve user data on upgrade.

### Repository Error Handling
**Known gap:** All AARRepository methods catch exceptions and log them, but do not propagate
errors to callers. The async executor pattern means callers have no feedback mechanism for
write failures. This is a known audit finding — error callbacks should be added in a future pass.

### Threading
- All DAO methods are synchronous (Room default)
- AARRepository wraps them in a single-thread executor for ordered background writes
- UI reads use LiveData (Room generates LiveData-returning DAO methods)
- WAL mode enables concurrent reads during background writes
